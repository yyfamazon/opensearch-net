    DataContractResolver

    A System.Text.Json contract resolver that makes the serializer honor the System.Runtime.Serialization attributes the client already uses (DataMemberAttribute, IgnoreDataMemberAttribute, DataContractAttribute), rather than requiring every model property to be re-annotated with [JsonPropertyName]/[JsonIgnore].

    This is foundational infrastructure for the Utf8Json → System.Text.Json migration (see GitHub issue #388): the client has ~3,700 [DataMember] attributes that must keep producing the same wire property names.

    • [DataMember(Name = "x")] → the JSON property name becomes x.
    • [IgnoreDataMember] → the property is not (de)serialized.
    • On a [DataContract] type, only members with [DataMember] are serialized (opt-in semantics).

    The client overwhelmingly declares these attributes on interfaces (for example ICharGroupTokenizer.MaxTokenLength carries [DataMember(Name = "max_token_length")] while the concrete CharGroupTokenizer.MaxTokenLength implements it implicitly with no attribute). The resolver therefore walks the concrete type's interface maps and inherits the attributes from the implemented interface property, mirroring how the vendored Utf8Json resolver (MetaType) resolved names.

    DoubleFormatConverter

    System.Text.Json converters for double and float that match the vendored Utf8Json number formatting (#388): an integral value in fixed notation is written with a trailing .0 (for example 2 → 2.0), which STJ's shortest form omits. Values already carrying a decimal point or an exponent (e.g. 1.5, 1E+20) are unchanged and already match. boost and many scored/float fields depend on this.

    STJ applies these to the corresponding nullable types (double?/float?) automatically via its built-in nullable wrapping.

    ObjectConverter

    A System.Text.Json converter for object-typed values (dynamic payloads such as script params, aggregation meta, _source fragments and Dictionary<string, object> values), part of the migration tracked by #388.

    By default System.Text.Json deserializes object to JsonElement, which neither matches the CLR shapes the client's code expects nor round-trips through the rest of the pipeline. This reproduces the vendored Utf8Json PrimitiveObjectFormatter mapping instead:

    • JSON object → Dictionary<TKey, TValue> of string to object.
    • JSON array → List<T> of object.
    • Integral number → long; other numbers → double.
    • String → string; true/false → bool; null → null.

    Writing delegates to the value's runtime type, so the shapes above serialize as expected.

    PolymorphicInterfaceConverter<TInterface>

    A reusable System.Text.Json converter for the client's many "single object with a type discriminator" polymorphic hierarchies (tokenizers, char filters, token filters, analyzers, normalizers, …), replacing the per-family hand-written Utf8Json *Formatter dispatchers as part of the migration tracked by #388.

    A concrete family only supplies the discriminator → concrete-type map; all read/write logic is shared here. This is the shape the converter generator emits per family, so adding a namespace is a table, not new dispatch code.

    • Write serializes the concrete runtime type. Because that type is not TInterface, this does not re-enter the converter.
    • Read buffers the object, reads the discriminator property, and deserializes into the mapped concrete type (property names resolved by DataContractResolver).

    StringEnumConverterFactory

    A System.Text.Json converter factory that serializes enums decorated with StringEnumAttribute as strings, honoring EnumMemberAttribute values (for example TokenChar.Whitespace → "whitespace").

    (Named with a Factory suffix to avoid colliding with Newtonsoft.Json.Converters.StringEnumConverter in consumers that import both namespaces.)

    This reproduces the vendored Utf8Json behavior (EnumFormatter with serializeByName: true): the built-in JsonStringEnumConverter emits the CLR member name and does not read EnumMemberAttribute on the target frameworks this client supports, so a dedicated converter is required for wire compatibility (#388).

    SystemTextJsonSerializer

    An IOpenSearchSerializer implementation backed by the Microsoft System.Text.Json library.

    This is the foundation for migrating the client away from the vendored, unmaintained Utf8Json serializer (see GitHub issue #388).
